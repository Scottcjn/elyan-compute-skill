# Contributing to Elyan Compute Skill

Thank you for your interest in contributing to the Elyan Compute Skill! This document provides guidelines and steps for contributing.

## How to Contribute

### Reporting Issues
- Check if the issue already exists in the [Issues](https://github.com/Scottcjn/elyan-compute-skill/issues) section
- Use a clear and descriptive title
- Provide steps to reproduce, expected behavior, and actual behavior
- Include environment details (OS, OpenClaw version, etc.)

### Suggesting Enhancements
- Open an issue with the "enhancement" label
- Clearly describe the feature and its use case
- If possible, provide examples or mockups

### Submitting Code Changes
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Ensure your code follows existing style and conventions
5. Test your changes
6. Commit with clear, descriptive messages
7. Push to your fork
8. Open a Pull Request

### Development Setup
1. Clone the repository
2. Install dependencies (if any)
3. Follow the README for setup instructions

## Code Style
- Follow the existing code style in the repository
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and small

## Pull Request Process
1. Update the README.md if needed
2. Update documentation (SKILL.md, etc.) if applicable
3. Your PR will be reviewed by maintainers
4. Address any feedback
5. Once approved, it will be merged
